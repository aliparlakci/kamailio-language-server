KSR Module & Method Completions
================================

Currently the extension only provides completions for KSR.pv.* methods.
The codebase uses 24 different KSR modules. The extension should provide
completions for all of them.

When the user types "KSR.", suggest all known modules:
  pv, pvx, hdr, htable, xlog, tm, tmx, sl, rr, dispatcher,
  sanity, siputils, maxfwd, enum, dialplan, sdpops, textopsx,
  corex, statistics, kx

When the user types "KSR.tm.", suggest all methods for that module:
  t_relay, t_on_failure, t_on_branch, t_check_trans,
  t_is_canceled, t_branch_replied, t_branch_timeout, t_send_reply

When the user types "KSR.", also suggest core methods:
  is_INVITE, is_ACK, is_BYE, is_CANCEL, is_NOTIFY, is_OPTIONS,
  is_method, is_method_in, seturi, set_drop

Data source: Build a comprehensive KSR KEMI method database from the
Kamailio documentation or scrape it from the KEMI exports.

Evidence from usage report:
  - 24 unique KSR modules used across the codebase
  - Currently zero completion support outside KSR.pv.*
