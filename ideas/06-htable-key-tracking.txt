Hash Table Key Tracking and Completions
========================================

KSR.htable.* methods use table names and key patterns. The extension
could track table names and key schemas to provide completions and
diagnostics.

Examples from usage (all use table "DestList"):
  KSR.htable.sht_get("DestList", callid + "::DnIdx")
  KSR.htable.sht_get("DestList", callid + "::DnNum")
  KSR.htable.sht_inc("DestList", callid + "::DnIdx")
  KSR.htable.sht_rm("DestList", KSR.pv.get("$ci") + "::DnIdx")
  KSR.htable.sht_seti("DestList", ...)
  KSR.htable.sht_sets("DestList", ...)

Also related PV access:
  KSR.pv.get("$shtinc(DestList=>...)")
  KSR.pv.get("$shtrecord(key)")
  KSR.pv.get("$shtrecord(value)")

Features:
  1. Completions for table names (first argument) based on workspace usage
  2. Completions for key suffixes ("::DnIdx", "::DnNum") based on patterns
  3. Hover info showing all access patterns for a table
  4. Diagnostics for mismatched table names (typo detection)
